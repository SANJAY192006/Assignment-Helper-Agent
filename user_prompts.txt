================================================================================
PART 1: PROFESSIONAL PROMPT SPECIFICATION PLAYBOOK
================================================================================

### Milestone 1: Frontend Redesign & UI Enhancement
* **Engineering Requirement**: Redesign the frontend from a basic interface into a premium, responsive glassmorphic UI using CSS variables and interactive layouts. Ensure page states are robust and query flows are fully interactive.
* **Refined Instruction**:
  > "Redesign the frontend UI from a simple layout to a modern, premium theme (using CSS variables, glassmorphic card designs, custom animations, and responsive layouts). Ensure the main interface handles query state transitions correctly and outputs detailed study guides."

### Milestone 2: Form Input Field Mapping & Validations
* **Engineering Requirement**: Fix issues in form validation and model parameter bindings so that user-submitted values mapped correctly to Gemini's schema.
* **Refined Instruction**:
  > "Fix form submission input processing. When a student selects a subject and inputs the topic and question, ensure the application captures and validates the payload correctly before dispatching the query request to the API."

### Milestone 3: AI Engine Integration & API Configuration
* **Engineering Requirement**: Securely configure and connect the Google Gemini API using externalized environment variables.
* **Refined Instruction**:
  > "Configure the Google Gemini API integration securely on the backend using the provided API key loaded via `.env` file variables. Use the `gemini-2.5-flash` model for structured JSON schema outputs."

### Milestone 4: Refactoring into Multi-Page Architecture
* **Engineering Requirement**: Refactor a single-page script into a multi-page model with clean layout separation and dynamic section styling.
* **Refined Instruction**:
  > "Refactor the application into a clean multi-page routing structure. Implement a dedicated login screen, route the main input details form onto its own page (`/form`), and render the generated 12-section learning guide on a separate full-width interactive workspace page (`/guide`). Ensure all card components render content clearly."

### Milestone 5: Authentication & Input Field Contrast Adjustments
* **Engineering Requirement**: Set up user authentication with hashed passwords, set up developer accounts, and improve form typography visibility.
* **Refined Instruction**:
  > "Implement secure user authentication with password hashing using Flask-Bcrypt. Configure default credentials (Username: `Kumar`, Password: `Kumar1234`) for dev-level access. Additionally, adjust typography settings inside text input fields and textareas to ensure clear contrast when the user enters text."

### Milestone 6: Guide View CSS Enhancements & PDF Rendering Fixes
* **Engineering Requirement**: Resolve client-side styling contrast issues, fix blank PDF exports caused by CSS variable mismatches, and prevent text truncation across page boundaries.
* **Refined Instruction**:
  > "Fix layout contrast and PDF printing bugs in the guide view. Resolve issues where: 1) input fields have poor visual visibility, 2) downloaded PDFs generated via `html2pdf.js` appear blank/empty due to incompatible dark-mode variables, 3) printed text contrast is faint, and 4) page breaks cut lines in half vertically. Ensure proper page-break-inside rules are set on section cards."

### Milestone 7: SQLite Database Connection & History Logs
* **Engineering Requirement**: Integrate a relational database using an ORM to persist assignments and user information, making it accessible through a dashboard view.
* **Refined Instruction**:
  > "Integrate an SQLite database with the Flask application using Flask-SQLAlchemy. Design relational schemas for User accounts and Assignment logs. Connect the generation endpoint to save all successfully generated guides linked to the active user's session ID and render them in a history log dashboard."

### Milestone 8: Documentation and Playbook Creation
* **Engineering Requirement**: Generate standard documentation and collect project logs.
* **Refined Instruction**:
  > "Generate comprehensive project documentation in the README.md detailing the backend system flow, technological architecture, database schemas, setup instructions, and troubleshooting steps. Export all iterative prompts into a structured log file."
